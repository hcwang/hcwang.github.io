发表于 2003-11-14 | 分类于 未分类 |

还记得开心的手机GPRS欠费事件吗？当时我的手机曾经一个半月欠费三万多元，全都是因为该死的GPRS。之所以欠费是因为我当时是以大客户的身份参与GPRS测试的，GPRS包月200元，但在测试期完成后，他们并没有通知我，而擅自改为了自由套餐，所以话费猛涨，差一点导致开心破产。

呵呵，怎么样？更详细的信息在这儿：http://www.bmcc.com.cn/index/svc/jfhb/，我的积分现在足以选择两张青鸟健身季卡了

我要去健身，太被中国移动感动了

MVP特写

刚才在小气的神的主页上看到了他写的MVP聚会的印象，没有想到小气的神记忆力这么好，都快过去一周了，他竟然还记得每个人的印象。



MVP特写：

开心：比WebLog上表现要温和一些，比自己的文字表现得要腼腆一些。适时的谦逊和温文尔雅，喜欢在轻松的气氛中达成自己的目标；当一个人乐于了解别人，并开纳朋友的胸怀，我想他也能获得更多的威信和信任。

波澜壮阔：沉静有力的握手和简单的话语，很容易让我想起自己熟悉的团队中的某个人。他会是一个在团队中拥有某种力量的人，以致于对于他的某个动作或是话语产生信赖和温暖的感觉。很多时候他们表现的是一种一贯的东西，尽管这一切，看起来非常的简单和朴实。

蝈蝈俊：开朗和敏捷的乐观者，某种程度上，他可能是一个不达目的不罢休的力量型实干家。很多时候他在用技术的眼光看待他周围的世界，当谈论到有关CSDN的论坛和我们对此惊奇的感觉时，他反而变得平静而一带而过，因为这些似乎是虚的，对一个实干家来说，过程和实现中的问题是他喜欢花费力气注意的。

蒋晟：不急于肯定和否定的现实主义，正如他对待VC++版主和Blogger的态度，正如他对待VC和VC.NET的态度，没有什么是完美和立即需要确定的，也没有什么是非黑即白的，对自己和技术的许多问题，他有自己的态度和看法，同样你不能很快改变他的看法。在没有搞清事实的情况下，你无法用时间表和激情说服他冒进或是盲从，在项目或团队中你会从他这里获得一种沉稳的信号。

小峰：正如他WebLog的副标题–读论文，读草案，想毕业。带着我们学生时代固有的热心、固执和快乐的气息。因为交谈的不多，所以从WebLog上了解他更多一些。率直的用词让他的WebLog轻而易举的充满个人主义。

一夜未睡

发表于 2003-11-14 | 分类于 未分类 |

终于在四天内结束了这一阶段的开发。

九月份在微软做了一个Biztalk 2004的Demo，这几天又是在做Biztalk的Demo。说实话，虽然原来改编过一本Biztalk的书籍，但对Biztalk了解的仍然不够，尤其是Biztalk 2004的变化非常大（内部消息，Biztalk 2004的汉化已经97%了，不过帮助仍然是英文版的），与VS.NET完全的集成式开发就让我非常的感兴趣。

通过这两次项目，我深深体会到，在一个团队的开发当中，需求分析是多么的重要。由于项目都比较小，而且工期非常紧，上次是两周，这次就四天。所以在前期只是初步定了接口，越做越发现问题，其实真正的编码往往都在最后几天。尤其这次的项目，真正的编码似乎只是这一夜而已。越做越发现原来的思路有问题，尤其团队成员间也只能通过开发的进度达到思路上的统一。

总体来说，我是幸福的

 

欢迎大家继续基于Office System 2003开发

发表于 2003-11-13 | 分类于 未分类 |

看了Moslem的新闻检索系统，感觉他做得实在太棒了，我相信这儿大部分人都是这样的感觉吧？

而且最主要的是他提供了很多这方面的资料，所以建议大家可以开发一下以下的几个产品：

  • 针对KB的检索
  • 针对MSDN Library的检索
  • 针对微软新闻组的检索

而且最好是把上面这三个检索放在同一个服务中，这样以后我们碰到相关的技术问题，就可以从IE或者Office当中得到解决。同时，为了便于客户端的开发，建议把业务逻辑做成类库，这样我们可以在上面分别包装成其它的Web服务，向客户端提供检索。

有人愿意做这样的工作吗？

另外，目前Visual Studio Tools For Office System也已经发布了，还有VSIP Extra SDK等，大家也可以基于这些技术进行一些开发，或者把自己的一些开发思路贡献出来，让博客堂能够引导开发新潮流

 

 

请注意博客堂帐号资源

发表于 2003-11-11 | 分类于 未分类 |

每天我都会收到至少五六封申请加入博客堂的邮件，目前这个数字正在逐步趋于稳定。

大家现在可能认为，在博客堂拥有一个帐号会是一个荣誉的标志，所以很多人都努力争取。但我发现有部分朋友，在没有开放帐号之前，会每天一封邮件，申请开通帐号，但在开通之后，却只是写一两个随笔在上面，而很少加入更新，更有甚者，很多人员把这儿当作了论坛或者自己问问题的场所，甚或是留言板。

希望大家能够珍重博客堂的帐号资源。在一个具有学习气氛的博客站点，我想人数保持在五六十人左右，是最正常的，所以我们可能在近期对帐号进行一些整理工作，帐号的封存采取自愿为主，如果您感觉实在没有时间进行内容更新，请通过邮件告诉我们。另外，对近期一个月没有更新过的博客将通过电子邮件进行通知，如果通知三次后，仍然无法联系，则自动进行封存。

请大家不要再通过电子邮件申请帐号了，目前，您可以通过参加博客堂网页设计大赛或者征文大赛赢得博客堂帐号。同时，如果您也可以在http://www.blogger.cn或者http://www.blogcn.com上申请帐号，我们会通过各种方式在网络上寻找比较适合博客堂气氛的朋友，邀请其加入博客堂。

博客堂找回密码功能

发表于 2003-11-08 | 分类于 未分类 |

已经有N多人告诉我说找不到自己的密码了，每次都是通过邮件或者MSN通知我，然后由我在数据库当中帮助他们寻找一下，再告诉他们。实在太累了，一点儿成就感都没有。

不过我不明白的是，为什么有这么多人记不住自己的密码？或者是现在的各种各样的应用太多了，每一个应用都一个密码，或者记起来是很麻烦，可是这里面竟然还有一些IT Pro也会忘记自己的密码。

现在好了，我仔细翻了一下.Text的工程，发现它有找回密码功能，就在你的地盘下面的“login.aspx”下面。比如我的就在:http://blog.joycode.com/joy/login.aspx，这里面有一个“通过邮件发送密码”的链接，点一下然后接收一下你在博客堂注册时使用的邮件，就可以看到密码即帐号了。
